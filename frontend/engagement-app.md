# Streamlit-приложение `engagement_app.py`

**Файл:** [`frontend/engagement_app.py`](../../frontend/engagement_app.py)


Данное приложение открывает веб-камеру, отправляет кадры на backend через WebSocket и отображает мультимодальные метрики и значение вовлечённости пользователя в реальном времени, предоставляя значения как напрямую, так и на графиках с течением времени соответственно.

---

## Запуск

### Предварительные требования

1. Запущен Redis (см. [../backend/deployment.md](../backend/deployment.md))
2. Запущен backend: `cd backend && uvicorn app.main:app --host 0.0.0.0 --port 8000`
3. Установлены frontend-зависимости: `cd frontend && pip install -r requirements.txt`
4. Подключена веб-камера (используется камера 0, т.е. первая веб-камера, определённая в системе)

### Команда

```bash
cd frontend
streamlit run engagement_app.py
```

Интерфейс откроется по адресу `http://localhost:8501`.

### Адреса backend

Управляются переменными окружения, грузятся через `python-dotenv` (`load_dotenv()` без аргументов, [engagement_app.py:14](../../frontend/engagement_app.py#L14)) – ищется `.env` в текущей директории запуска. При запуске из `frontend/` это `frontend/.env`. Системные переменные окружения имеют приоритет над `.env`.

| Переменная | Default | Описание |
|------------|---------|----------|
| `BACKEND_WS_URL` | `ws://localhost:8000` | WebSocket-адрес для `/ws/rooms/{room_id}/stream` |
| `BACKEND_HTTP_URL` | `http://localhost:8000` | HTTP-адрес для `GET /health` |

Подробнее: [environment.md](environment.md).

### Проверка backend

При загрузке страницы и далее каждые **10 секунд** (`HEALTH_CHECK_INTERVAL`) фронтенд опрашивает `GET /health`. Если backend недоступен, то отображается предупреждение с адресом, элементы управления камерой не показываются.

Источник: [`check_backend_health`](../../frontend/engagement_app.py#L134).

## Управление

### Запуск веб-камеры

Кнопка **"▶️ Запустить"**:
1. Устанавливает `webcam_running = True` и `needs_reset = True` в `session_state`.
2. После rerun:
   - открывается `cv2.VideoCapture(0)` (разрешение **320×240**, **15 FPS**, см. [`engagement_app.py:342-344`](../../frontend/engagement_app.py#L342-L344));
   - устанавливается WS-соединение к комнате `engagement-app` с именем `engagement-user` (если ещё не подключено);
   - очищается история всех графиков и счётчики кадров.

### Остановка

Кнопка **"⏹️ Стоп"** ставит `webcam_running = False`, после rerun:
- `cap.release()` – камера освобождается;
- WS-соединение закрывается через `api_client.disconnect()`;
- последние данные на графиках остаются на экране.

### Параметры обработки

В `engagement_app.py` параметры backend (пороги EAR, HPE, эмоций) **не меняются через UI**. Они задаются через **env-переменные при запуске backend**. Для тюнинга в рантайме используйте [`tools/param_testing_app.py`](param-testing-tool.md).

## Текущие показатели (левая колонка)

### Компоненты вовлечённости

Три блока в одну строку, отображающие `attention_state` + компонентные значения (частные score-показатели):

| Блок | Источник | Формат |
|------|----------|--------|
| `Eye:` | `result.ear.attention_state` + `engagement.components.eye_score` | `Eye: Alert (0.85)` |
| `HPE:` | `result.head_pose.attention_state` + `engagement.components.head_pose_score` | `HPE: Attentive (0.80)` |
| `Emo:` | `engagement.components.emotion_score` | `Emo: 0.72` |

> При отсутствии соответствующих данных блок очищается.

### Общая вовлечённость

```
Вовлечённость: High (78%) →
```

Формат `{level} ({score%}) {trend_icon}`. Иконки тренда:

| `trend` | Иконка |
|---------|--------|
| `rising` | `↑` |
| `falling` | `↓` |
| `stable` | `→` |

### Эмоция

```
Happiness (уверенность: 0.87)
```

Если backend не нашёл лицо в кадре, то отображается предупреждение: `Лицо не обнаружено`.

### Углы головы

```
P: 5.2° · Y: -2.1° · R: 1.0°
```

При отсутствии HPE-данных: `P: – · Y: – · R: –`.

## Графики (правая колонка)

Обновляются каждые `CHART_UPDATE_INTERVAL = 15` кадров (~1 секунда при 15 FPS) для уменьшения нагрузки на Streamlit.

### Распределение эмоций

Donut-диаграмма Plotly. Накапливает историю эмоций из `result.emotion`, **до 100 последних** кадров (`emotion_history = deque(maxlen=100)`).

### Положение головы

Линейный график 3 линий (Pitch – красный, Yaw – зелёный, Roll – синий). Показывает **последние 30 точек** из `head_pose_history`. Пунктирная горизонталь на `0` представляет собой нейтральное положение.

### Eye Aspect Ratio

Линейный график `avg_ear`. Пунктирная красная линия на пороге `EAR_THRESHOLD` (default `0.25`, читается из env). Последние 30 точек.

> Env-переменная `EAR_THRESHOLD` для фронтенда **не управляет** реальным порогом моргания на backend, она изменяет только положение красной линии на графике (порога на графике). Реальный порог настраивается в backend через собственную переменную (см. [../backend/environment-variables.md](../backend/environment-variables.md)).

После остановки камеры графики не обнуляются: на экране остаются последние полученные данные. История очищается только при следующем нажатии "▶️ Запустить" (`needs_reset = True`).

## Поведение в типичных сценариях

| Сценарий | Что происходит |
|----------|----------------|
| Backend недоступен на старте | Появляется предупреждение, кнопки камеры скрыты |
| Backend упал во время сессии | `api_client.send_frame` возвращает `(None, [])`, метрики очищаются, видео показывает сырой кадр без аннотаций (прямоугольников и текста метрик от пайплайна) |
| Лицо вышло из кадра | `results: []`, метрики компонентов очищаются, эмоция = "Лицо не обнаружено" |
| Веб-камера не открылась | Предупреждение `Не удалось открыть веб-камеру`, `webcam_running = False` |

## Жизненный цикл WebSocket

```
"▶️ Запустить" нажата
        │
        ▼
api_client.connect(room_id="engagement-app", name="engagement-user")
        │
        ▼
цикл: read_frame → send_frame → отображение
        │
        ▼
(WS соединение хранится в session_state.api_client._ws)
        │
        ▼
"⏹️ Стоп" нажата → webcam_running = False
        │
        ▼
finally: cap.release(); если webcam_running == False → api_client.disconnect()
```

При rerun Streamlit (например, из-за health-check) WS **не закрывается**, т.е. соединение переиспользуется. Закрытие происходит только при явной остановке.

Reconnect выполняется автоматически в `EngagementAPIClient.send_frame`: одна попытка реконнекта при ошибке передачи. Если она не удалась, то кадр пропускается.
