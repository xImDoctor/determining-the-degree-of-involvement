# `tools/param_testing_app.py` – утилита тюнинга параметров

**Файл:** [`frontend/tools/param_testing_app.py`](../../frontend/tools/param_testing_app.py)

Утилита для подбора параметров пайплайна **без участия backend**: модули ML импортируются напрямую и работают локально. Нужна, чтобы быстро проверять, как изменения порогов EAR/HPE/эмоций влияют на детекцию и калибровать соответствующие параметры.

---

## Чем отличается от основного приложения

| Признак | `engagement_app.py` | `tools/param_testing_app.py` |
|---------|---------------------|------------------------------|
| Источник обработки | Backend через WebSocket | Локальный (импорт `app.services.video_processing`) |
| Требует запущенный backend | Да | **Нет** |
| Требует Redis | Да (через backend) | **Нет** |
| Параметры в UI | Нет | Да, слайдеры в боковой панели |
| Разрешение веб-камеры | 320×240, 15 FPS | 640×480, native FPS |

---

## Запуск

```bash
cd frontend
streamlit run tools/param_testing_app.py
```

### Зависимости

Импортирует backend через `sys.path.append`:

```python
# frontend/tools/param_testing_app.py:14
sys.path.append(str(Path(__file__).resolve().parent.parent.parent / "backend"))
```

Это значит, что:
- `backend/` должен лежать рядом с `frontend/` в той же иерархии.
- Должны быть установлены backend-зависимости (mediapipe, torch, emotiefflib, opencv-python).

Проще всего поставить весь backend через `cd backend && pip install -e ".[dev]"`, тогда frontend подхватит модули.

## Вкладки

### 1. "🎬 Загрузка и обработка"

Назначение: прогнать видеофайл через пайплайн и сохранить результат.

**Шаги:**
1. Перетащить видео или выбрать через `Browse Files`.
2. Проверить превью и метаданные (разрешение, длительность, FPS).
3. Нажать **"🚀 Начать распознавание эмоций"**.
4. Дождаться окончания обработки (прогресс-бар + список текущих эмоций).
5. Скачать результат через `📥 Скачать обработанное видео`.

**Поддерживаемые форматы:** MP4, AVI, MOV, MKV, WebM, WMV.
**Максимальный размер файла:** 100 МБ (`MAX_FILE_SIZE = 100 * 1024 * 1024`, см. [param_testing_app.py:43](../../frontend/tools/param_testing_app.py#L43)).

Выходное видео сохраняется во временную директорию ОС (`tempfile.gettempdir()`) с именем `emotion_detected_{input_name}_{timestamp}.mp4`.

### 2. "📷 Веб-камера"

Назначение: проверить параметры на живом видео.

**Шаги:**
1. Настроить параметры в боковой панели.
2. Нажать **"🎬 Запустить веб-камеру"**.
3. Изменения параметров слайдеров применяются "на лету".
4. Нажать **"⏹️ Остановить веб-камеру"** для завершения.

### 3. "❓ Справка"

FAQ + типовые проблемы. Не содержит interactive-функций, только статичный текст, *актуальный на момент первоначальной разработки данного инструмента*.

## Параметры в боковой панели

### Детекция лиц

| Слайдер | Параметр | Default | Range |
|---------|----------|---------|-------|
| Минимальная уверенность детекции | `min_detection_confidence` | 0.50 | 0.00–1.00 |

### Распознавание эмоций

| Слайдер | Параметр | Default | Range |
|---------|----------|---------|-------|
| Размер окна сглаживания | `window_size` | 15 | 3–30 |
| Порог уверенности | `confidence_threshold` | 0.55 | 0.00–1.00 |
| Порог амбивалентности | `ambiguity_threshold` | 0.15 | 0.00–1.00 |

### Дополнительные модули (опциональные)

| Чекбокс | Параметр | Default |
|---------|----------|---------|
| Включить Eye Aspect Ratio | `enable_ear` | False |
| Включить Head Pose Estimation | `enable_head_pose` | False |

При включении EAR появляются дополнительные слайдеры:

| Слайдер | Параметр | Default | Range |
|---------|----------|---------|-------|
| Порог EAR | `ear_threshold` | 0.25 | 0.10–0.40 |
| Кадров для моргания | `consec_frames` | 3 | 1–10 |

> HPE при включении использует пороги из `settings` (env). Слайдеров для порогов attention_state в боковой панели **нет**.

### Общие настройки

- `flip_h` – отзеркалить кадр по горизонтали (для веб-камеры удобно).
- `show_preview` – показывать превью кадров во время offline-обработки.

### Визуализация

Управляют **только** аннотациями на UI, не меняют параметры моделей и хранятся отдельно от `backend_params`:

| Чекбокс | Эффект |
|---------|--------|
| Показывать FPS | Накладывает FPS-счётчик на кадр |
| Показывать эмоции | Текстовый блок с эмоциями под видео |
| Показывать данные EAR | EAR-данные в текстовом блоке |
| Показывать данные HPE | HPE-данные в текстовом блоке |

> **Важно**: параметры применяются к инструменту, для применения их ко всей системе (бэкенду) необходимо задать новые параметры в .env-файле бэкенда (см. [../backend/environment-variables.md](../backend/environment-variables.md)).

## Очистка ресурсов

При нажатии "⏹️ Остановить веб-камеру":

```python
# param_testing_app.py:1124-1131
if not st.session_state.get('webcam_running', True):
    if st.session_state.webcam_detector:
        if hasattr(...face_detector, 'close'):
            ...face_detector.close()
        if hasattr(...emotion_recognizer, 'reset'):
            ...emotion_recognizer.reset()
    st.session_state.webcam_detector = None
    st.session_state.prev_webcam_params = None
```

Зачистка делается только при явной остановке.

## Когда использовать

- Подбор `confidence_threshold` для конкретной освещённости.
- Проверка, как `consec_frames` влияет на пропуск/ложные моргания.
- Калибровка `ear_threshold` под особенности глаз пользователя.
- Прогон тестового видео и сравнение с эталоном.

После определения хороших параметров их прописывают в `.env` для backend (см. [../backend/environment-variables.md](../backend/environment-variables.md)) – основное приложение `engagement_app.py` подхватит новые значения при следующем запуске backend.
